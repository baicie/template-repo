# 流水账检测

## 触发场景

用户说"流水账检测"、"检查流水账"、"平淡检测"、"流水账"、"没有冲突"。

## 流水账特征

符合以下全部特征则为流水账：

1. **无冲突**：只有事件罗列，没有矛盾冲突
2. **无情绪**：角色的行为没有情绪支撑
3. **无期待**：事件按时间顺序平铺，没有悬念

## 检测维度

### 1. 事件罗列检测

检查是否有连续3段以上的纯事件描写，格式为：
```
他做了A，然后做了B，接着做了C。
```

### 2. 情绪空白检测

检查角色的行为是否有内心支撑：
- 是否有心理描写？
- 行为是否有情感动机？
- 是否只是"他做了..."的被动句式？

### 3. 期待感缺失检测

检查结尾是否有钩子：
- 是否抛出悬念？
- 是否留有疑问？
- 是否有未完成的冲突？

### 4. 节奏单一点检测

检查是否所有段落都是同一节奏：
- 句子长度是否都差不多？
- 是否都是短句或都是长句？
- 是否有节奏变化？

## 评分标准

- 0-30分：严重流水账，几乎无价值
- 31-60分：中等流水账，有问题区域
- 61-80分：轻微流水账，个别平淡
- 81-100分：非流水账，节奏良好

## 改进公式

**原文**：事件A → 事件B → 事件C

**改写**：事件A → 意外/冲突 → 情绪反应 → 悬念钩子

### 改进示例

❌ 流水账：
```
他起床，洗漱，吃早饭，出门，坐地铁，上班。
```

✅ 改进：
```
他被闹钟吵醒时就知道今天会很难熬。

昨晚的方案又被驳回了。领导那句"你再想想"在他脑子里转了一夜。

出门时钱包落在玄关。地铁上差点挤丢了手机。

但这些都不重要——今天，他决定摊牌。
```

