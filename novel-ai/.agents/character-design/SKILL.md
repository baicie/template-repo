# Character Design — 角色塑造

## 流程顺序

塑造角色应按以下顺序进行：

```
1. protagonist.md  → 设计主角
2. antagonist.md   → 设计核心反派
3. supporting.md   → 设计重要配角
4. character-web.md → 构建关系网
5. speaking-style.md → 提炼话风
```

## 1. 主角设定 (protagonist)

主角是整个故事的核心，主角的设定决定故事的走向。

### 模板

```markdown
# 主角设定

## 基础信息
- 姓名：{{name}}
- 性别：{{gender}}
- 年龄（初登场）：{{age}}
- 身份/职业：{{occupation}}

## 外在形象
【外貌】3-5 句话，有记忆点
【气质】从什么角度描写
【标志物】最有辨识度的特征

## 内在设定
【性格核心】1-2 个词
【性格成因】这个性格是怎么形成的
【行为模式】遇到事情的典型反应（3 个场景）
【底线/原则】绝对不能触碰的红线
【缺陷/弱点】推动剧情的关键

## 成长弧线
【起点】故事开始时的状态
【转折点】性格/认知发生重大变化的事件
【终点】成长后的状态

## 关系网络
【挚爱】{{relationship1}}
【挚敌】{{relationship2}}
【羁绊】{{relationship3}}

## 写作要点
【话风】说话特点
【行动风格】打斗、做事的风格
【心理描写】侧重哪种情感
【专属词汇】口头禅、高频用词

## 写作禁忌
【绝对避免】这个角色不会做的事
【语气把控】避免写成 XX
```

### 主角设计的核心要点

- **记忆点要突出**：给读者一个能记住的画面（如：腰间别着的断剑、左手的黑色手套）
- **缺陷是剧情引擎**：完美的主角无聊，有缺陷才有成长空间
- **成长弧线清晰**：读者要能看到主角的成长
- **爽点要埋好**：主角的逆袭点要提前设计好

## 2. 反派设定 (antagonist)

反派的质量决定主角的高度，好的反派让故事更精彩。

### 模板

```markdown
# 反派设定

## 基础信息
- 姓名：{{name}}
- 身份：{{identity}}
- 与主角的关系：{{relationship_to_protagonist}}

## 角色定位
【威胁等级】对主角的威胁程度
【出现阶段】第几卷/什么阶段开始正式对抗

## 外在形象
【外貌】有压迫感/辨识度的描写
【气场】给人的第一印象

## 内在设定
【性格核心】性格特点
【行为动机】为什么要和主角作对
【行事风格】阴险/霸道/算计型？

## 能力设定
【实力】相对主角的优势
【特殊能力】让主角头疼的能力
【弱点】主角最终战胜他的关键

## 与主角的关系线
【对立起源】为什么成为敌人
【多次交锋】对抗升级的过程
【最终对决】决战的条件和方式

## 写作禁忌
【绝对避免】把这个反派写成 XX
```

### 反派设计的核心要点

- **动机合理**：反派的动机要让读者能理解
- **有独特个性**：不是工具人，有自己的追求和逻辑
- **实力要在线**：让读者相信他确实能威胁到主角
- **有弱点可利用**：主角最终战胜他要有迹可循

## 3. 配角设定 (supporting)

配角是丰富故事的填充，让世界更真实。

```markdown
# 配角设定

## 基础信息
- 姓名：{{name}}
- 身份：{{identity}}
- 与主角的关系：{{relationship}}

## 功能定位
【角色类型】导师型/朋友型/暗恋型/对手型？
【出场频率】高/中/低
【核心作用】推动什么剧情

## 角色设定
【性格】性格特点
【话风】说话风格
【特色】最有记忆点的特征

## 成长弧线（如有）
配角自己的成长线

## 与其他角色的关系
- 与主角
- 与其他配角
```

### 配角设计的核心要点

- **功能明确**：每个配角都要有存在的理由
- **个性鲜明**：即使是配角也要有辨识度
- **关系真实**：配角之间的关系要有真实感
- **戏份适度**：不要让配角抢了主角的戏份

## 4. 角色关系网 (character-web)

构建所有角色之间的关系网络。

```markdown
# 角色关系网

## 核心关系
| 角色 A | 关系 | 角色 B | 关系描述 |
|--------|------|--------|---------|
| 主角 | 对立 | 反派 | 主要矛盾方 |
| 主角 | 挚友 | 配角 A |  |
| ... | ... | ... | ... |

## 势力阵营
- 主角阵营：成员列表
- 反派阵营：成员列表
- 中立势力：成员列表

## 关系变化
- 哪些关系会在故事中发生变化
- 变化的触发条件
```

## 5. 话风提炼 (speaking-style)

为每个重要角色提炼专属的话风，让对话更有辨识度。

```markdown
# 话风提炼

## 主角话风
【语速】快/慢/适中
【用词】书面/口语/古风/网络
【句式】短句/长句/混合
【口头禅】{{catchphrase}}
【特色表达】{{signature_expression}}

## 对话示例
A：（对话场景）
A：「{{dialogue_sample}}」

## 话风描写片段
描写这个角色说话时的整体感觉
```

## 角色设计原则

1. **主角要让人有代入感**：读者能理解主角的选择
2. **反派要让人又恨又服**：既讨厌又觉得厉害
3. **配角要有独特功能**：不重复，不冗余
4. **关系要有张力**：对立中有合作，合作中有矛盾
5. **设定要可执行**：设计出来的角色要能写进情节里

## 输出存储

所有角色输出存入 `books/{书名}/00-characters.md`，按以下结构组织：

```markdown
# {{book_title}} — 角色档案

## 主角
## 核心反派
## 重要配角
## 角色关系网
## 话风提炼
```
