# Switch Book — 书籍切换

## 用途

当用户需要切换到另一本书写作时，执行以下步骤。

## 切换流程

### 步骤 1：确认目标书籍

```
用户：「切换到星际迷途」
用户：「我要写另一本书」
用户：「帮我看看逆天改命第 237 章」
```

### 步骤 2：执行切换

执行以下操作：

```
1. 更新 configs/active-book.md
2. 更新 .cursor/rules/current-book.md
3. 确认切换成功
```

### 步骤 3：确认切换成功

向用户确认：

```
已切换到「{书名}」
- 类型：{类型}
- 进度：第 {N} 章
- 状态：{连载中/完结/暂停}
- 路径：books/{书名}/

可以开始写作了。
```

## 涉及的文件

### configs/active-book.md

```markdown
# 当前激活的书籍

active_book: {书名}
book_path: ./books/{书名}/
```

### .cursor/rules/current-book.md

```markdown
# 当前书籍上下文

## 激活的书：{书名}

请在所有写作任务中，以 `books/{书名}/` 为项目根目录。

当前进度：
- 总字数：约 {X} 万字
- 当前：第 {N} 章 / {卷名}
- 状态：{连载中/暂停/完结}

阅读顺序：
1. 先读 meta.md 了解作品定位
2. 再读 00-context.md 了解当前状态
3. 根据需要读取角色/世界观/章节文件
```

## 切换时的操作

### 必须执行

1. 读取 `BOOKS.md` 确认目标书籍存在
2. 读取 `books/{目标书名}/meta.md` 获取基本信息
3. 更新 `configs/active-book.md`
4. 更新 `.cursor/rules/current-book.md`
5. 向用户确认切换成功

### 建议执行

1. 读取 `books/{目标书名}/00-context.md` 预览当前状态
2. 告知用户当前写作进度和关键信息

## 多本书管理原则

1. **始终只有一个激活书籍**：当前书籍是唯一的
2. **切换时完整切换**：不要混淆两本书的上下文
3. **进度定期更新**：每次写作后更新 00-context.md
4. **统一命名规范**：章节文件使用一致的命名规则

## 快速切换命令

| 用户说 | 执行 |
|--------|------|
| 「切换到 XX」 | 切换到指定书 |
| 「当前在写哪本」 | 读取 active-book.md 告知 |
| 「列一下所有书」 | 读取 BOOKS.md 列出 |
| 「新建一本书」 | 引导创建新书目录结构 |
