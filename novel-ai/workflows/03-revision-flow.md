# 修订润色流程

## 修订类型

### 小修（每章写完后）

- 润色对话
- 消除语病
- 检查一致性
- 调整字数

### 中修（每卷写完后）

- 通读全卷
- 检查前后矛盾
- 节奏审视
- 伏笔检查

### 大修（初稿完成后）

- 全书一致性检查
- 角色弧线审视
- 主线逻辑审视
- 整体节奏调整

---

## 流程一：章节润色

### 适用场景

章节草稿完成后，想快速提升文字质量。

### 流程

```
1. 确定润色类型（对话/动作/心理/整体）
2. 读取 configs/writing-rules.md 的禁用词
3. 应用对应的润色提示词
4. 对比原文和润色稿
5. 作者确认最终版本
```

### 常用润色提示词

| 润色类型 | 提示词文件 |
|---------|---------|
| 对话润色 | `prompts/writing/02-dialogue-polish.md` |
| 战斗场景 | `prompts/writing/03-action-scene.md` |
| 情感场景 | `prompts/writing/04-emotion-scene.md` |
| 整体打磨 | `prompts/writing/06-prose-polish.md` |

## 流程二：一致性检查

### 适用场景

写完一个章节或段落后，检查是否与已有设定矛盾。

### 流程

```
1. 读取待检查章节
2. 读取相关设定文件
3. 应用 consistency-check 模板
4. 逐条分析问题
5. 给出修改建议
6. 更新伏笔追踪
```

### 检查清单

```
力量体系
[] 角色实力等级是否匹配
[] 能力描述是否与设定一致
[] 战斗结果是否符合实力逻辑

角色行为
[] 对话风格是否与设定一致
[] 行为是否符合性格
[] 心理描写是否合理

地理/时间
[] 场景描写是否合理
[] 时间线是否连贯

逻辑
[] 情节推进是否合理
[] 是否有伏笔未回收
[] 金手指使用是否合理
```

## 流程三：节奏审视

### 适用场景

写完一卷或全书后，检查节奏是否合适。

### 流程

```
1. 读取 configs/writing-rules.md 的节奏规范
2. 通读全卷/全书
3. 标记高潮点
4. 分析节奏曲线
5. 给出调整建议
```

### 节奏检查点

```
爽点分布
[] 每章是否有小高潮
[] 是否有连续 3 章以上的平淡期
[] 高潮是否足够密集

章节类型分布
[] 日常章不超过 3 章连续
[] 战斗章后有缓冲
[] 转折章分布均匀

情感节奏
[] 情感升温是否循序渐进
[] 是否有足够的情感释放点
[] 读者情绪是否有起伏
```

## 流程四：伏笔追踪

### 适用场景

全书或全卷完成后，确保伏笔全部回收。

### 流程

```
1. 读取 00-context.md 的伏笔追踪表
2. 逐条检查伏笔状态
3. 标记已回收的伏笔
4. 标记未回收但计划内的伏笔
5. 标记遗漏的伏笔
```

### 伏笔追踪表示例

```markdown
| 伏笔 | 埋下章节 | 计划回收 | 实际回收 | 状态 |
|------|---------|---------|---------|------|
| 玄铁令来历 | 第 12 章 | 第 40 章 | 第 38 章 | ✅ 已回收 |
| 主角身世 | 第 8 章 | 第 100 章 | — | ⏳ 待回收 |
| 系统新功能 | 第 210 章 | — | — | 📌 提及但未展开 |
```

## 修订优先级

| 优先级 | 类型 | 时机 | 重要程度 |
|--------|------|------|---------|
| P0 | 严重逻辑矛盾 | 立即修复 | 必须 |
| P1 | 前后设定矛盾 | 下章写前修复 | 必须 |
| P2 | 语病/错别字 | 润色时修复 | 重要 |
| P3 | 节奏问题 | 中修时调整 | 重要 |
| P4 | 文笔提升 | 大修时优化 | 建议 |
