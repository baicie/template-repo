# 单章写作流程

## 核心流程

```
确认上下文 → 填充模板 → 写作 → 审阅 → 更新上下文
```

---

## 步骤一：确认上下文

### 必须读取

1. `books/{书名}/00-context.md`
   - 当前章节号
   - 本卷概要
   - 角色状态
   - 章节上下文（承接/任务）

2. `books/{书名}/00-outline.md`
   - 本章大纲节点

3. `books/{书名}/00-characters.md`
   - 本章出场角色

4. `books/{书名}/00-world.md`
   - 本章涉及的世界观要点

### 可选读取

- 前 1-3 章内容（确保衔接）
- 相关角色详细设定

---

## 步骤二：填充写作模板

使用 `prompts/writing/01-chapter-draft.md`：

```markdown
# 第 {{chapter_number}} 章「{{chapter_title}}」写作

## 章节信息
- 所属卷：{{volume_name}}
- 章节类型：{{chapter_type}}
- 目标字数：{{word_count}} 字
- 章节视角：{{viewpoint}}

## 大纲节点
{{chapter_outline}}

## 出场角色
{{active_characters}}
【重点】{{focus_character}} 的心理和行动

## 世界观要点
{{world_rules_applicable}}

## 文风要求
- 整体风格：{{overall_tone}}
- 段落节奏：{{pacing}}
- 重点描写：{{emphasis}}

## 前文衔接
{{previous_chapter_summary}}

【开头要求】承上启下的方式
【结尾要求】留钩子的方式

## 写作指令
请按照以上设定，创作第 {{chapter_number}} 章完整正文。
```

---

## 步骤三：写作

### 输出规范

- 直接输出正文，不加任何解释
- 章节标题：`## 第 X 章`
- 字数控制在目标范围内
- 开头承上启下
- 结尾留悬念

### 常见情况处理

| 情况 | 处理方式 |
|------|---------|
| 卡文 | 先给出 2-3 个方向让作者选择 |
| 上下文不足 | 询问作者补充信息 |
| 发现矛盾 | 指出矛盾，等待作者决策 |
| 字数超标 | 先完成，事后可删减 |

---

## 步骤四：审阅

### 自我检查

```
[] 是否承接了上文
[] 是否有小高潮
[] 结尾是否留了钩子
[] 字数是否合适
[] 有无明显语病
[] 有无禁用词
```

### 一致性检查（如需要）

使用 `.agents/skills/consistency-check/` 进行检查。

---

## 步骤五：更新上下文

章节完成后，更新 `00-context.md`：

### 必须更新

- 当前章节号：`第 X 章`
- 字数统计
- 角色状态变化
- 情节进展

### 可能更新

- 伏笔回收情况
- 势力变化
- 物品变化

### 示例更新

```markdown
## 章节上下文

### 承接第 236 章结尾
> 萧寒立于庭中，手中捏着那张密信...

### 本章（第 237 章）任务
- 情节目标：主角决定提前入京
- 核心爽点：越级战斗秒杀刺客
- 出场人物：萧寒、刺客三人组

---

## 章节上下文（更新后）

### 承接第 237 章结尾  ✅ 已完成
> 萧寒拔出刺客身上的令牌，上面刻着「赵」字...

### 第 238 章待写
- 情节目标：主角进京，遭遇太子党
- 伏笔回收：赵无极身份暴露
```

---

## 快速命令

```
写章节：        「写第 X 章」
只看草稿：      「先给我一个大纲」
润色对话：      「润色这段对话」
快速润色：      「润色整章」
检查一致性：    「检查本章和前文是否矛盾」
```
