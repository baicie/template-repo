# 章节自检清单

> 每章写作前后的检查项。确保质量、稳定、不崩。

---

## 写前预检

### 1. 情节确认

```
□ 承接上文结尾（复制粘贴上章最后200字）
□ 本章要解决什么问题？
□ 本章要引出什么新问题？
□ 本章出场角色是否都已入库 tracking/character-state.json？
```

### 2. 爽点确认

```
□ 本章有没有爽点？
□ 爽点前面有足够压抑吗？
□ 爽点有没有提前铺垫（不是凭空出现）？
□ 爽点后有没有新期待？
```

### 3. 设定确认（防幻觉）

```
□ 主角境界是否符合 tracking/character-state.json？
□ 没有让角色做出与设定矛盾的事？
□ 没有篡改力量体系？
□ 没有让已死亡的角色出场？
```

### 4. Strand 确认

```
□ Quest（主线）是否推进？
□ Fire（感情线）是否断档太久？
□ Constellation（世界观）是否揭示了新信息？
```

### 5. 债务确认

```
□ 是否有债务即将到期？
□ 本章是否需要兑现某个债务？
□ 是否埋下了新的债务（伏笔）？
```

---

## 写后复核

### 1. 字数检查

```
□ 目标字数：X
□ 实际字数：X
□ 差值在 ±10% 以内？
```

### 2. 结构检查

```
□ 开头是否承上启下？
□ 中间是否有节奏变化？
□ 结尾是否有钩子？
```

### 3. 逻辑检查

```
□ 事件因果是否成立？
□ 角色行为是否符合性格？
□ 对话是否符合人设？
```

### 4. 水分检查

```
□ 有没有废话/水文？
□ 有没有无意义的日常描写？
□ 有没有可以删除的段落？
```

### 5. 钩子检查

```
□ 章节结尾是否有钩子？
□ 钩子是否够强？
□ 读者会想知道下一章发生什么吗？
```

### 6. 更新检查

```
□ tracking/plot-tracker.json 是否更新？
□ tracking/character-state.json 是否更新？（如有变化）
□ tracking/timeline.json 是否记录新事件？
□ tracking/hooks.json 是否记录新钩子？
□ tracking/debts.json 是否记录新债务？
□ 00-context.md 是否更新章节上下文？
```

---

## 问题修复优先级

| 优先级 | 问题类型 | 处理方式 |
|--------|---------|---------|
| P0 | 设定矛盾/战力崩坏 | 必须立即修复 |
| P1 | 人设OOC | 修复后再发布 |
| P2 | 流水账/节奏拖沓 | 压缩或重写 |
| P3 | 钩子不够强 | 增强结尾 |
| P4 | 文笔润色 | 可在润色期处理 |

---

## 发布前最终确认

```
□ 所有追踪文件已更新
□ 无P0/P1级问题
□ 章节字数在合理范围
□ 结尾有强钩子
□ 上下文更新后逻辑自洽
□ AI痕迹已处理（如需发布）
```
