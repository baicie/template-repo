# 网文写作方法论

> 融合乌贼式世界观、起点商业化、唐神式爽点节奏三大体系。用于指导 AI 写作时理解网文核心逻辑。

---

## 一、网文不是"写好文笔"，而是持续制造期待

网文的核心是 **让读者持续想看下一章**。

核心问题链：

```
读者为什么点开？
读者为什么看完第一章？
读者为什么愿意追到十万字？
读者为什么一直追更？
```

答案不在于文笔，而在于：

```
主角想要什么？
主角缺什么？
谁阻止他？
他靠什么逆转？
每一章给读者什么新期待？
```

## 二、三大派系方法论

### 派系一：爱潜水的乌贼 — 世界观驱动型

**核心**：先搭一个有规则、有秘密、有探索感的世界，再让升级体系从世界观里自然长出来。

**适合类型**：玄幻、悬疑、诡异、奇幻

**关键方法**：

#### 1）世界观不是百科，而是"规则 + 欲望 + 危险"

❌ 错误：
```
这个世界有九大境界、三十六洞天、七十二福地...
```

✅ 正确：先让主角遇到具体问题

```
不要先讲世界多宏大
先让主角遇到一个具体问题

他为什么必须变强？
这个世界有什么危险？
某个规则违反后会有什么代价？
```

#### 2）升级体系要是"剧情发动机"

普通升级流：
```
炼体 → 筑基 → 金丹 → 元婴
```

发动机式升级：
```
升级方式：主角如何变强？
升级代价：变强会失去什么？
升级资源：靠什么获取？
升级门槛：谁掌握资源？
升级风险：失败会怎样？
升级反馈：变强后能解决什么问题？
```

#### 3）类型融合有主菜和调料

```
主菜（核心类型）：玄幻升级 / 都市异能 / 仙侠修真
调料（融合元素）：克苏鲁 / 赛博朋克 / 民俗诡异 / 游戏面板

例：主菜是玄幻升级，调料是民俗诡异
→ 剧情仍围绕宗门、战斗、资源展开
→ 诡异元素让升级过程更有新鲜感
```

### 派系二：起点创作学堂 — 商业工程型

**核心**：商业网文要工程化，开篇、人设、大纲、细纲、正文都要服务读者期待和持续追读。

**适合类型**：都市、穿越、重生、系统流

**关键方法**：

#### 1）开篇四件事（前3章必须完成）

```
身份：主角是谁
困境：主角现在有什么麻烦
优势：主角有什么特殊能力
期待：接下来有什么期待
```

常见男频爽文开篇结构：

```
第一章：主角被压迫，核心矛盾爆发
第二章：金手指/机会出现，初步反击
第三章：爽点兑现，更大危机/目标出现
```

#### 2）大纲三层结构

```
一级大纲：整本书讲什么
二级大纲：每一卷讲什么
三级细纲：每5-10章发生什么
```

**实用做法**（新手友好）：

```
先定前30章
再定第一个大高潮
再定第一个地图结束点
```

#### 3）人设卡不只是档案，而是剧情发动机

❌ 不够：
```
姓名、年龄、外貌、性格、身份
```

✅ 足够：
```
欲望：他想要什么？
恐惧：他怕失去什么？
能力：他靠什么解决问题？
缺陷：他为什么会犯错？
关系：他和主角有什么利益冲突？
剧情作用：他推动哪条线？
```

---

### 派系三：唐家三少 / 恩赐解脱 — 爽点节奏型

**核心**：男频爽文靠稳定连载、清晰目标、连续升级、阶段爽点和职业化自律。

**适合类型**：玄幻、修仙、热血

**关键方法**：

#### 1）男频爽文基本循环

```
压迫 → 目标 → 行动 → 受阻 → 反击 → 爽点兑现 → 更大目标
```

玄幻升级流示例：

```
被羞辱
发现资源/金手指
修炼突破
小试牛刀
打脸敌人
获得新资源
进入更大地图
遇到更强敌人
```

地图扩张路径：
```
家族线 → 城池线 → 宗门线 → 王朝线 → 大陆线 → 上界线
```

#### 2）爽点必须提前埋

```
前面被嘲笑废物 → 后面突破打脸才爽
前面资源被抢 → 后面夺回才爽
前面亲人被欺负 → 后面保护才爽
前面敌人高高在上 → 后面让他失态才爽
```

爽点公式：

```
读者期待的情绪 × 足够长的压抑 × 合理的逆转方式 = 爽点
```

#### 3）升级必须有"阶段反馈"

❌ 错误：修炼三天，境界从炼气一重变成二重，读者无感

✅ 正确：升级要让读者看到具体剧情权限变化

```
升级后能打败之前打不过的人
升级后能进入之前进不去的地方
升级后能获得之前拿不到的资源
升级后能知道之前不知道的秘密
升级后能保护之前保护不了的人
```

#### 4）稳定输出比灵感更重要

```
每天写1500-3000字
先坚持30天
再优化节奏和质量

比起"今天灵感爆发写一万，后面断更一周"
稳定输出更重要
```

## 三、最实用写作公式

### 好题材

```
好题材 = 熟悉类型 + 新鲜设定
```

### 好开篇

```
好开篇 = 主角困境 + 明确目标 + 特殊优势 + 第一个冲突
```

### 好章节

```
好章节 = 推进剧情 + 提供爽点 + 留下期待
```

### 好长篇

```
好长篇 = 世界规则 + 升级体系 + 地图扩张 + 持续反馈
```

### 好作者

```
好作者 = 稳定输出 + 持续复盘 + 不断优化读者体验
```

## 四、完整开书流程

### 第一步：确定一句话卖点

```
格式：主角 + 困境 + 特殊能力 + 成长方向 + 类型期待

示例：一个被废的少年，获得能吞噬神明残魂的黑鼎，从此走上弑神修仙路。
```

### 第二步：设计主角

```
主角是谁？
他缺什么？
他想要什么？
他为什么非赢不可？
他靠什么和别人不同？
他最大的缺陷是什么？
```

### 第三步：设计金手指

```
金手指能力：
使用条件：
使用代价：
成长方式：
隐藏秘密：
会带来什么敌人：
```

### 第四步：设计前三章

```
第一章：主角处境 + 矛盾爆发
第二章：金手指/机会出现 + 初步反击
第三章：爽点兑现 + 更大危机出现
```

### 第五步：设计前30章

```
前10章：主角觉醒能力，解决第一个小敌人
10-20章：进入更大冲突，获得关键资源
20-30章：第一个大高潮，完成身份/实力跃迁
```

前30章必须有一次明显阶段胜利：
```
从废物变成可被重视的人
从底层进入宗门
从被追杀变成反杀
从无名小卒变成小范围出名
```

## 五、章节写作清单

每章必须确认：

```
【情节推进】
□ 主角做了什么？
□ 解决了什么问题？
□ 遇到了什么新问题？

【爽点/期待】
□ 本章有什么爽点？
□ 铺垫了什么？
□ 结尾留下了什么期待？

【信息增量】
□ 读者知道了什么新信息？
□ 主角获得了什么新能力/资源？
□ 有什么新角色/势力出场？

【钩子】
□ 章节结尾有什么钩子？
□ 下章会解决什么？

【字数】
□ 目标字数：X
□ 实际字数：X
```
