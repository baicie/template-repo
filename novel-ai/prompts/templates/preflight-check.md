# 写作前预检

> 每次写作前必须执行。确保 AI 不会遗忘设定、不会产生幻觉。

## 预检清单

```
【1. 确认当前章节】
- 章节号：
- 承接上文结尾：

【2. 大纲即法律】
- 本章对应的大纲节点：
- 是否有偏离大纲的计划？
  □ 无偏离
  □ 有偏离（说明：）

【3. 设定即物理】
- 主角当前境界：
- 主角当前位置：
- 涉及的角色（按 tracking/character-state.json）：
- 涉及的世界规则（按 00-world.md）：

【4. 发明需识别 - 本章可能新增】
- 新角色出场：□ 是 / □ 否
- 新关系建立：□ 是 / □ 否
- 新事件发生：□ 是 / □ 否
- 新设定揭示：□ 是 / □ 否

【5. 防幻觉检查】
□ 没有让角色做出与设定矛盾的事
□ 没有修改力量体系
□ 没有让已死亡的角色出场
□ 没有让角色突然获得未铺垫的能力

【6. 债务到期检查】
当前活跃债务：
- debt-XXX：剩余 X 章
- debt-YYY：剩余 X 章

是否有债务即将到期（≤3章）？
□ 无
□ 有：

【7. 钩子延续检查】
上章结尾钩子：
- hook-XXX：

【8. Strand 检查】
上一章 Strand 分布：
- Quest：X%
- Fire：X%
- Constellation：X%

是否需要调整？
□ 无需调整
□ 需要加入 Fire
□ 需要加入 Constellation
□ 需要加入 Quest
```

## 预检输出

预检完成后，输出：

```
# 预检报告 - 第 X 章

## 章节信息
- 章节号：X
- 大纲节点：...
- 偏离情况：无/有（说明）

## 上下文确认
- 主角状态：炼气境 X 重，在 XX
- 出场角色：XX、XX
- 世界规则：...

## 设定一致性
【通过/警告】无设定冲突

## 发明登记
- 新增角色：X 个（待入库）
- 新增关系：X 个（待入库）
- 新增事件：X 个（待入库）

## 债务状态
- 即将到期：debt-XXX（剩余 X 章）

## Strand 分布
- Quest：X%
- Fire：X%
- Constellation：X%

## 预检结论
【通过/需调整】
```

**只有预检通过后才能开始写作。**
