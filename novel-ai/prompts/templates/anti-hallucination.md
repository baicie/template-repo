# 防幻觉三定律

> 来自 webnovel-writer 的核心理念。AI 在长篇创作中最怕两件事：**遗忘**（前后矛盾）和**幻觉**（擅自发明）。这三定律是所有写作任务的最高约束。

## 第一定律：大纲即法律

**遵循大纲，不擅自发挥。**

- 每章必须先读取本章对应的大纲节点
- 如果要偏离大纲，必须标注【偏离大纲】并说明理由
- 偏离大纲超过 3 处，必须暂停并等待作者确认
- 禁止为了"让剧情更好看"而擅自改变主线走向

## 第二定律：设定即物理

**遵守设定，不自相矛盾。**

- 每章必须先确认相关角色、势力、世界规则的当前状态
- 禁止让角色做出与设定矛盾的行为
- 禁止修改已确立的力量体系
- 禁止让已死亡的角色复活
- 禁止让角色突然获得未铺垫的能力

## 第三定律：发明需识别

**新实体必须入库管理。**

- 新出场的重要角色必须记录到 `tracking/character-state.json`
- 新建立的关系必须记录到 `tracking/relationships.json`
- 新发生的事件必须记录到 `tracking/timeline.json`
- 新设定的世界规则必须记录到 `00-world.md`
- 所有新增内容必须在章节结尾的更新报告中说明

## 执行检查清单

每次写作前：

```
【大纲即法律】
[] 本章对应的大纲节点已读取
[] 没有偏离主线
[] 偏离已标注并说明

【设定即物理】
[] 角色状态与 tracking/*.json 一致
[] 力量体系没有被篡改
[] 没有让角色做设定外的事

【发明需识别】
[] 新角色已入库
[] 新关系已入库
[] 新事件已入库
[] 已在更新报告中说明
```

每次写作后：

```
【发明需识别 - 报告】
[] 新增角色：X 个（已入库/未入库）
[] 新增关系：X 个（已入库/未入库）
[] 新增事件：X 个（已入库/未入库）
[] 新增设定：X 处（已入库/未入库）
```
