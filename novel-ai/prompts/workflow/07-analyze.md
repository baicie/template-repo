# 综合验证

```markdown
# 综合验证

## 任务
对已完成的章节进行七个维度的质量验证。

## 已有信息
- 书名：{{book_title}}
- 待验证章节：第 {{chapter_number}} 章

## 验证框架

### 维度 1：合规性
- 是否符合 `constitution.md` 中的创作宪法？
- 是否遵循了质量底线？

### 维度 2：一致性
- 逻辑一致性：情节逻辑是否自洽？
- 人物一致性：角色行为是否符合设定？
- 世界观一致性：力量体系、世界规则是否前后一致？

### 维度 3：完整性
- 需求覆盖：本章是否完成了既定目标？
- 任务完成：本章任务清单是否完成？
- 伏笔铺设：本章是否埋下必要伏笔？

### 维度 4：质量性
- 文字质量：语句是否通顺、用词是否精准？
- 结构质量：开头、结尾、高潮是否到位？
- 节奏质量：是否有节奏变化、是否拖沓？

### 维度 5：创新性
- 亮点识别：本章有哪些创新的点？
- 特色识别：是否形成了独特风格？
- 突破识别：是否有超越预期的表现？

### 维度 6：可读性
- 流畅度：阅读是否流畅？
- 吸引力：开头是否吸引人？
- 共鸣度：读者是否能产生共鸣？

### 维度 7：可行性
- 进度可行性：当前进度是否合理？
- 资源可行性：是否有足够的创作资源？
- 风险识别：是否有潜在风险？

## 验证清单

```
【合规性】
[] 符合创作宪法
[] 符合质量底线

【一致性】
[] 情节逻辑自洽
[] 角色行为一致
[] 世界观前后一致

【完整性】
[] 章节目标完成
[] 伏笔铺设到位

【质量性】
[] 文字通顺
[] 结构完整
[] 节奏合适

【创新性】
[] 有亮点
[] 有特色

【可读性】
[] 流畅
[] 吸引人

【可行性】
[] 进度合理
[] 无明显风险
```
