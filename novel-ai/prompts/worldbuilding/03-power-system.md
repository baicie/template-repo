# 力量体系设计

## 任务
为「{{book_title}}」（{{genre}}）设计一套完整的力量体系。

## 已有信息
- 书名：{{book_title}}
- 类型：{{genre}}
- 基本创意：{{basic_idea}}

## 体系命名
{{system_name}}（体系名称，要有辨识度）

## 等级设定

| 等级 | 名称 | 标志性特征 | 能力范围 | 突破条件 |
|------|------|-----------|---------|---------|
| 1 | {{level_1}} | {{feature}} | {{ability}} | {{condition}} |
| 2 | {{level_2}} | | | |
| 3 | {{level_3}} | | | |
| 4 | {{level_4}} | | | |
| 5 | {{level_5}} | | | |
| 6 | {{level_6}} | | | |
| 7 | {{level_7}} | | | |

## 力量来源
- 力量从哪里来？
- 如何修炼/觉醒？
- 是否有天赋限制？

## 等级制约
- 高等级对低等级的压制关系
- 越级战斗的可能性和代价
- 等级压制在战斗中的表现

## 特殊路径
- 是否有旁门左道？
- 速成方法的代价是什么？
- 特殊血脉/体质的影响？

## 体系特点
- 这个体系最独特的地方是什么？
- 如何让读者容易记忆？
- 哪些设定可以用来设计爽点？

## 设计要点
1. 名称要有辨识度，避免直接用"一级、二级"
2. 等级数量适中，7-9 级为宜
3. 每个等级有标志性特征，读者一听就能想象
4. 升级条件明确，主角奋斗的目标
5. 升级代价合理，代价越大爽感越强
